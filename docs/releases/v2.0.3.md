## v2.0.3 — Open-Meteo 실 API 연동

### 주요 변경
- v2.0.2까지 `WeatherRepository`가 반환하던 mock 데이터를 모두 제거하고, Open-Meteo의 공개 API(키 불필요)로 실제 날씨를 가져오도록 했습니다.
- 도시명 검색은 [Open-Meteo Geocoding](https://open-meteo.com/en/docs/geocoding-api)으로 위경도를 조회한 뒤, Forecast/Air-Quality API에 위경도를 전달합니다.

### 코드
- `app/src/main/java/.../data/api/OpenMeteoModels.kt` 신규: Geocoding/Forecast/AirQuality 응답 데이터 클래스.
- `app/src/main/java/.../data/api/WeatherApi.kt`: 기존 OpenWeatherMap 인터페이스를 제거하고, `GeocodingApi`/`ForecastApi`/`AirQualityApi` 3종 Retrofit 인터페이스로 교체.
- `app/src/main/java/.../data/repository/WeatherRepository.kt`: mock 함수 제거. 실 API 응답을 기존 `CurrentWeatherResponse`/`ForecastResponse`/`FineDustInfo` 모델로 매핑. WMO 코드 → 내부 conditionId 매핑 헬퍼 포함. minSdk 24 호환을 위해 `java.time` 대신 `SimpleDateFormat`/`Calendar`/`TimeZone` 사용.

### 매니페스트 / 빌드
- `AndroidManifest.xml`에 `INTERNET`, `ACCESS_NETWORK_STATE` 권한을 추가했습니다.
- `app/build.gradle.kts`의 versionCode 기본값 202→203, versionName 기본값 "2.0.2"→"2.0.3"으로 bump.

### 릴리즈 노트 / 문서
- `play_store/release_notes/v2.0.3.txt` (ko-KR / en-US 다국어) 작성.
- `CHANGELOG.md`에 v2.0.3 섹션 추가.

### 검증
- 로컬: `KEYSTORE_PATH`, `STORE_PASSWORD`, `KEY_ALIAS`, `KEY_PASSWORD`, `ANDROID_HOME` 환경변수와 함께 `./gradlew bundleRelease`로 서명된 AAB 산출 확인.
- 기존 `ExampleRobolectricTest`, `GreetingScreenshotTest`는 네트워크 호출이 Robolectric에서 실패하므로 ViewModel은 `WeatherUiState.Error`로 진입합니다. UI 캡처 자체는 깨지지 않지만, 향후 fake `WeatherRepository`를 주입할 수 있도록 ViewModel을 리팩토링하는 것이 자연스러운 다음 단계입니다.

### Play Store 영향
- 데이터 안전 설문: 이번 빌드부터는 사용자 입력 검색어가 외부 서버(Open-Meteo)로 전송되므로, "검색기록" 카테고리를 **수집·필수·앱 기능용·공유 없음**으로 표시하는 것이 정확합니다. 단, 사용자 식별 정보와는 연결되지 않습니다(앱이 사용자 식별 정보를 보유하지 않음). 위치 좌표는 사용자 GPS에서 얻은 것이 아니라 검색어를 Geocoding으로 변환한 값입니다.
- 권한 변경(인터넷 추가)은 Play Console에서 자동 감지됩니다.
