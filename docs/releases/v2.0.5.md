## v2.0.5 — 도시 검색 보정 및 위치 기반 선택 개선

### 주요 변경
- **서울/한글 도시 검색 보정**: Open-Meteo Geocoding API가 `서울` 같은 한글 검색어에 결과를 반환하지 않는 문제를 확인하고, 주요 한국 도시명을 API 검색 가능한 영문명으로 정규화했습니다.
- **추천 도시 제거**: 하단 추천 도시 칩을 제거해 고정 도시 버튼에 의존하지 않도록 정리했습니다.
- **위치 기반 선택 추가**: 사용자가 직접 누를 때만 대략적인 위치 권한을 요청하고, 현재 위치 기준의 날씨/예보/대기질을 조회하는 버튼을 추가했습니다.
- **위치 선택 저장**: 현재 위치 선택 시 마지막 위치 좌표와 표시명을 기기 내부에 저장해 새로고침과 다음 실행에서도 같은 위치 기준으로 조회합니다.

### 코드 / 리소스 변경
- `app/src/main/AndroidManifest.xml` [MODIFY]: 선택형 현재 위치 조회를 위한 `ACCESS_COARSE_LOCATION` 권한 추가.
- `app/src/main/java/com/jeiel/zephyr_sky/data/repository/WeatherRepository.kt` [MODIFY]: 도시명 정규화와 좌표 직접 조회 경로 추가.
- `app/src/main/java/com/jeiel/zephyr_sky/ui/weather/DeviceLocationResolver.kt` [NEW]: Android `LocationManager`/`Geocoder` 기반 현재 위치 해석 유틸리티 추가.
- `app/src/main/java/com/jeiel/zephyr_sky/ui/weather/WeatherScreen.kt` [MODIFY]: 추천 도시 칩 제거 및 현재 위치 선택 버튼 추가.
- `app/src/main/java/com/jeiel/zephyr_sky/ui/weather/WeatherViewModel.kt` [MODIFY]: 위치 좌표 저장, 복원, 새로고침 흐름 추가.
- `app/src/test/java/com/jeiel/zephyr_sky/WeatherRepositoryTest.kt` [NEW]: 한국 도시명 정규화 단위 테스트 추가.
- `app/build.gradle.kts` [MODIFY]: default versionName `"2.0.4"` -> `"2.0.5"`, versionCode `204` -> `205` 로 bump.
