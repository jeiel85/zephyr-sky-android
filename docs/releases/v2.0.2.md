## v2.0.2 — Android 패키지 식별자 정렬

### 주요 변경 사항
- Android Native 리뉴얼(v2.0.0) 과정에서 템플릿 기본값 그대로 남아 있던 `com.example` 패키지 식별자를 `com.jeiel.zephyr_sky`로 되돌렸습니다.
- 이 식별자는 v1.x Flutter 빌드와 동일한 값입니다. Play Store에 동일 앱 업데이트로 올리려면 `applicationId`가 일치해야 하므로, v2.0.1 릴리즈 노트의 후속 조치(실제 패키지 ID 확인) 항목을 처리한 릴리즈입니다.

### 수정 사항
- `app/src/main`, `app/src/test`, `app/src/androidTest`의 Kotlin 소스 14개를 `com/example/`에서 `com/jeiel/zephyr_sky/`로 이동하고, 모든 `package`/`import` 선언과 fully-qualified 참조를 새 패키지로 갱신했습니다.
- `app/build.gradle.kts`의 `namespace`와 `applicationId`를 `com.jeiel.zephyr_sky`로 변경하고, versionCode 기본값 201→202, versionName 기본값 "2.0.1"→"2.0.2"로 bump했습니다.
- `ExampleInstrumentedTest`의 패키지명 단언값을 새 식별자로 갱신했습니다.

### 문서 / 빌드 / 배포
- GitHub Actions `Android CI`의 에뮬레이터 스모크 테스트 명령을 `adb shell am start -W -n com.jeiel.zephyr_sky/com.jeiel.zephyr_sky.MainActivity` 기준으로 갱신했습니다.
- `CHANGELOG.md`에 `v2.0.2 - 2026-05-22` 섹션을 추가했습니다.
- Play Store용 다국어 릴리즈 노트를 `play_store/release_notes/v2.0.2.txt`에 작성했습니다.

### 검증
- 로컬: `ANDROID_HOME=C:\Users\jeiel\AppData\Local\Android\Sdk` 지정 후 `.\gradlew.bat test` 성공.
- 로컬: `ANDROID_HOME=C:\Users\jeiel\AppData\Local\Android\Sdk` 지정 후 `.\gradlew.bat assembleDebug --no-configuration-cache` 성공.
- CI: 태그 푸시 후 `Android CI`와 `Android Build & Release` 워크플로우 결과는 릴리즈 시점에 별도로 확인이 필요합니다.

### 설치 또는 업데이트 참고 사항
- `applicationId`가 v2.0.0/v2.0.1의 `com.example`에서 `com.jeiel.zephyr_sky`로 변경되었습니다. 기기에 v2.0.0 또는 v2.0.1 디버그 빌드가 설치되어 있다면 별개 앱으로 인식되므로 기존 버전을 제거한 뒤 설치하시는 것을 권장합니다.
- Play Store에 첫 업로드를 진행하는 경우, 릴리즈 서명 키와 패키지명이 Play Console에 등록한 앱의 값과 일치하는지 한 번 더 확인이 필요합니다.
